HTML

  • The head element contains information, called metadata, about the webpage.
  • The body element represents the visible contect shown to the user.
  • The ul, ol and li elements represent unordered lists, ordered lists, and list items.
  • The src attribute is the most important in img elements because it defines the location, or source, of the image file.
  • The alt attribute contains a text string that describes the appearance and functionality of the image.
  • The a element stands for anchor; it creates links to the same webpage or other webpages.
  • The href attribute points to the URL for the link.
  • Header appears at the top of the page and usually contains a navigation menu and/or introductory content.
  • When using headings, it's important to use heading tags in order from h1-h6 to make the page accessible to users who access the page using screen readers or other devices.
  • Footer appears at the bottom of the page and usually contains the author, copuright, contact, sitemap, and navigation.
  • Section elements are semantic elements that define a section in a document that contains themantic content.
  • ID attribute is a way to identify elements and select them.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git revert: allows you to revert back to an earlier version of code
  • git add -A: the add command adds modifications in the current working branch to the staging area; the -A flag indicates that we want to add all changes
  • git commit: a commit is created with a message that describes what the commit contains
  • git pull origin main: pulls down any changes from the base branch
  • git push origin branch-name: pushes the changes to the remote branch
